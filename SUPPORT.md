# Support

## Getting Help

1. Read `README.md` and `docs/` first.
2. Run `termlm doctor --strict` and collect output.
3. Check `termlm status --verbose` and daemon logs.
4. Open a GitHub issue with repro details.

## Where to Ask What

- Usage/setup questions: GitHub Discussions (Q&A category)
- Bug reports: GitHub Issues
- Security vulnerabilities: GitHub Security Advisory draft (`/security/advisories/new`)

## Information to Include

- OS/arch and zsh version
- install path (`with-models` or `no-models`)
- config snippet (redacted)
- exact commands and outputs
- expected vs actual behavior
